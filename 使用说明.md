# 航班管理系统使用说明

## 1. 项目简介
本系统使用 C 语言开发，采用以下实现方式：

- 使用内核双向链表（`kernel_list.h`）管理用户、航班、预订数据
- 使用结构体封装业务对象（`User`、`Flight`、`Booking`）
- 按功能模块拆分源文件，便于维护与扩展

系统包含用户端和管理员端功能，并实现了价格、折扣、退票手续费、联合航班等扩展能力。

## 2. 目录结构

```text
include/
  flight_system.h      # 公共结构体、常量、函数声明
  kernel_list.h        # 内核链表实现
src/
  main.c               # 程序入口
  fs_data.c            # 全局链表与初始化/清理
  fs_utils.c           # 输入输出与工具函数
  fs_user.c            # 用户注册/登录/用户菜单
  fs_flight.c          # 航班查询/录入/更新/删除/显示
  fs_booking.c         # 预订/退订/预订链表管理
  fs_admin.c           # 管理员菜单与用户管理
```

## 3. 编译与运行

### 3.1 Windows + MinGW（推荐）
在项目根目录执行：

```powershell
gcc -std=gnu11 -Wall -Wextra -Iinclude src/*.c -o flight_system.exe
.\flight_system.exe
```

### 3.2 Linux/macOS

```bash
gcc -std=gnu11 -Wall -Wextra -Iinclude src/*.c -o flight_system
./flight_system
```

```text

```

## 6. 用户功能
用户登录后可进入用户菜单：

- `1 航班查询`：按航班号、出发地+目的地、出发时间精确查询
- `2 模糊查询`：
  - 价格段查询：输入目标价格，系统按 `±200` 区间匹配经济舱票价
  - 起飞时间段查询：输入时间，系统按 `±30` 分钟匹配
- `3 航班预订`：
  - 输入航班号与航司（联合航班需指定航司）
  - 选择舱位（经济/商务/头等）
  - 计算规则：`实付 = 舱位票价 * 用户折扣 + 燃油费 + 机建费`
- `4 我的预订`：查看当前用户全部已预订航班
- `5 取消预订`：
  - 退还票面可退部分
  - 燃油费、机建费不退
  - 按手续费比例扣减（当前为 10%）

## 7. 管理员功能
管理员登录后可进入管理员菜单：

- `1 航班录入`
  - 录入航班号、出发地、目的地、出发/到达时间、座位数、各舱位价格
  - 航司管理员默认只能录入自己航司的航班
- `2 航班删除`
  - 按航班号（可配合航司）删除
  - 删除时同步释放该航班下全部预订记录
- `3 航班更新`
  - 可更新航班号、出发地、目的地、时间、座位、价格
  - 若总座位改小且小于已预订人数，会按“最新购票先移除”回收预订
- `4 航班显示`
  - 展示可管理范围内的航班
- `5 用户管理`
  - 添加用户
  - 删除用户（会自动清理该用户全部预订并回补座位）
  - 更新用户信息
  - 查看用户列表

## 8. 业务规则

- 舱位类型：`1经济舱 2商务舱 3头等舱`
- 用户组折扣：
  - `normal`：1.00
  - `student`：0.90
  - `soldier`：0.85
  - `elder`：0.80
- 固定费用：
  - 燃油费：`50.0`
  - 机建费：`60.0`
- 退票手续费率：`10%`
- 航班可重复航班号（用于联合航班），以“航班号 + 航司”区分具体航班

## 9. 模块封装说明

- 数据对象定义统一在 `include/flight_system.h`
- 链表操作统一基于 `include/kernel_list.h`
- 每个功能模块单独源文件实现，避免 `main.c` 业务臃肿
- 跨模块通过函数声明调用，减少耦合

## 10. 常见问题

### 10.1 中文显示问题
已在程序中做了 Windows 控制台 Unicode 输出兼容。若个别终端仍显示异常，请关闭并重新打开终端后再运行程序。

### 10.2 为什么查询同航班号有多条
这是联合航班设计，可能由不同航司共同运营，价格和座位独立管理。
